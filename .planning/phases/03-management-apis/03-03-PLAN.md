<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| client→`/api/email-services*` | 非可信 service config 和管理动作进入邮箱服务管理端点 |
| emailservices API→provider test/outlook import helpers | 测试连接和 Outlook 批量导入会触达外部服务或批量写入 |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-03-01 | I | list/detail/full 端点 | mitigate | 明确分离 filtered config 与 `/full` 返回；handler 测试锁定 list/detail 不回传敏感字段，只暴露 `has_*` 和当前页面需要的 hints。 |
| T-03-03-02 | T | service create/update/reorder | mitigate | 使用 typed request structs 和稳定 priority/order 规则，拒绝未定义 service_type 和无效排序输入，避免把无效配置写入共享表。 |
| T-03-03-03 | D | Outlook batch import/delete | mitigate | 在 service 层批量校验输入、限制空导入/重复项，并通过测试锁定当前批量结果语义，避免一次坏数据破坏整页管理状态。 |
</threat_model>
