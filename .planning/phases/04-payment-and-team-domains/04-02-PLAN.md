<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| browser/static-js→team handlers | 页面把 team_id、membership_id、invite payload 和 task 触发请求送入 Go team slice |
| team task admission→shared jobs/websocket runtime | Team accepted-task 需要复用现有 jobs/task websocket，而不是创建第二套运行时 |
| team adapter→upstream Team API | discovery/sync/invite/membership actions 可能访问外部 Team 上游接口 |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-02-01 | D | active scope admission | mitigate | 在 tasks/service tests 中锁定 `active_scope_key` 去重与冲突路径，避免同一 team/owner 被并发写任务破坏。 |
| T-04-02-02 | T | membership actions and invite flows | mitigate | 复用当前 status/source 规则和前置条件校验，handler 只做 decode，业务守卫留在 service/tasks 层。 |
| T-04-02-03 | R | Team accepted-task observability | mitigate | 继续返回 `task_uuid/ws_channel` 并复用现有 `/api/ws/task/{task_uuid}`，保证 discovery/sync/invite 有统一可追踪日志和完成态。 |
</threat_model>
