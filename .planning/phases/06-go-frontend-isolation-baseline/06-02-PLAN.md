<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| browser → Go admin login/session | Go 新增页面入口开始处理 operator cookie 和登录跳转 |
| adminui static mount → chi router | 错误挂载会让 `/admin/static/*` 与现有 route/middleware 冲突 |
| Go admin frontend → existing `/api/*` | 新页面入口不能改变已有 API 或 websocket 所有权边界 |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-06-02-01 | S | admin auth cookie | mitigate | 复刻当前 Python 基线的 HMAC/cookie 比较语义，并通过 auth tests 锁定登录、登出、`next` 跳转行为。 |
| T-06-02-02 | T | chi static mount | mitigate | 依照 Context7 的 chi 文档，避免 `RedirectSlashes` + `http.FileServer` 组合；router tests 直接验证 `/admin/static/*` 可访问且不抢占 `/api/*`。 |
| T-06-02-03 | D | route ownership boundary | mitigate | `router.go` / `main.go` 测试同时覆盖 `/healthz`、`/api/*`、`/api/ws/*` 与 `/admin/*`，防止 admin wiring 让现有 API 路径消失或漂移。 |
</threat_model>

<risk_and_rollback>
## 风险

- Go 侧若引入新的强制 auth 配置而不兼容现有输入，会把“基线独立启动”变成新的部署阻塞。
- static mount 如果处理不好，会出现 `/admin/static/*` 404、错误重定向，或干扰现有 chi 路由行为。

## 回滚点

- 如果 adminui wiring 造成路由冲突，只回滚 `backend-go/internal/http/router.go`、`backend-go/cmd/api/main.go` 以及本计划新增的 Go admin handler/auth wiring；保留 06-01 的 copied asset tree，legacy frontend 继续作为稳定 fallback。
- 若 auth/config 方案不稳，优先回退到最小兼容输入语义，不在 Phase 6 扩展额外安全能力。
</risk_and_rollback>
