## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Native runner result -> executor persistence | Untrusted runtime output crosses into durable account writes here. |
| Persisted account -> auto-upload dispatcher | Sensitive account/token fields are reused for CPA/Sub2API/TM side effects after persistence. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-03-01 | T | executor persistence chain | mitigate | Lock Python-compatible account field semantics in boundary and persistence tests before changing implementation, including partial-failure persistence behavior. |
| T-02-03-02 | I | auto-upload side effects | mitigate | Keep upload dispatch inside the executor-owned ordered chain and continue using Go account merge/update semantics so writebacks cannot bypass contract enforcement. |
| T-02-03-03 | R | upload writeback asymmetry | mitigate | Encode CPA/Sub2API-vs-TM asymmetry in `auto_upload_dispatcher_test.go` and executor tests so later changes cannot silently turn TM into a mutating writeback path. |
</threat_model>
