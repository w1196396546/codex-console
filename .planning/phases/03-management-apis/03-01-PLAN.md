## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| client→`/api/accounts*` | 非可信 query/body 进入高权限账户管理端点 |
| accounts service→uploader targets | 手动上传会调用外部 CPA/Sub2API/TM 服务 |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-01-01 | T | `backend-go/internal/accounts/http/handlers.go` | mitigate | 使用 typed request structs 和显式 query normalization，仅接受当前页面已使用的 filter/body 字段并在 handler 层做边界校验。 |
| T-03-01-02 | I | 账户详情/tokens/cookies 端点 | mitigate | 只返回 Python oracle 已暴露的字段集合，新增 handler/e2e 测试锁定字段名，避免把 `extra_data` 或内部写回元数据整包外泄。 |
| T-03-01-03 | D | 手动上传动作 | mitigate | 复用现有 `uploader` sender 的超时与结果构造逻辑，不在 handler 中手写外部 HTTP 请求，也不在本计划中放大批量上传并发。 |
</threat_model>
