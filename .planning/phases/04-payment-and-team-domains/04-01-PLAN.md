<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| browser/static-js→payment handlers | 运营端页面把账号 ID、绑卡数据、任务动作和 subscription 同步请求送入 Go payment slice |
| payment service→accounts persistence | payment slice 会读写 session token、cookies、subscription 元数据 |
| payment adapter→browser/third-party payment runtime | 高风险绑卡/checkout 探测可能访问浏览器自动化或第三方支付状态接口 |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-01-01 | T | `bind_card_tasks` state machine | mitigate | 在 service/repository tests 中锁定合法状态流转与 `paid_pending_sync`/`completed`/`failed` 分支，避免被任意请求篡改到非法状态。 |
| T-04-01-02 | I | session/bootstrap and card payloads | mitigate | 明确保留现有“表格只显示 masked 值、错误只走 detail、敏感值不回显”的 handler contract，不把 PAN/CVC/session secrets 暴露到列表响应。 |
| T-04-01-03 | D | browser/third-party bind execution | mitigate | 对高风险执行步骤引入显式 adapter seam 和超时/错误归一化，避免 handler 直接阻塞或把 Python fallback 分散到多个入口。 |
</threat_model>
