# Phase 1 Route and Client Parity Matrix

## Decision Traceability

- **D-01:** Python route behavior remains the compatibility oracle for every not-yet-migrated surface.
- **D-02:** Path, method, payload/status fields, websocket semantics, and critical side effects are all part of parity.
- **D-03:** Existing Go registration/task APIs, websocket streams, account-listing baseline, jobs, and uploader persistence are treated as baseline.
- **D-04:** Phase 1 freezes the migration delta and safety rails; it does not re-implement existing Go baseline behavior.
- **D-05:** Current templates and static/js files are first-class compatibility consumers and are named explicitly below.
