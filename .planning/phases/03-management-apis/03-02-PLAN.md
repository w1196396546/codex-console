<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| client→`/api/settings*` | 非可信配置写入、数据库管理和代理测试请求进入高权限端点 |
| settings API→proxy/database helpers | 动态代理测试和数据库导入/清理会调用外部网络或高影响内部操作 |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-02-01 | T | settings/proxy write handlers | mitigate | 所有 settings/proxy/database 请求都使用 typed request structs、范围校验与当前确认语义；禁止接受未在 Python oracle 中出现的自由键写入。 |
| T-03-02-02 | I | `/api/settings` 聚合响应 | mitigate | 聚合响应只返回当前页面需要的字段和值来源；敏感值延续现有“显示配置，不扩散凭据”的模式，避免在非 full 端点回显 secrets。 |
| T-03-02-03 | D | dynamic proxy test / database import | mitigate | 在 service 层固定请求方法、超时和导入前备份/确认流程，避免 handler 直接执行无界 IO 或无确认的 destructive 操作。 |
</threat_model>
