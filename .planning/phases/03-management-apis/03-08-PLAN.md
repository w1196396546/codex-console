<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| client→`/api/sub2api-services/upload` | 未受信的 account IDs、priority、concurrency 与 service_id 进入 Go API |
| Go API→PostgreSQL accounts rows | direct-upload 会读取账号令牌并回写 `sub2api_uploaded` 状态 |
| Go API→Sub2API external service | 上传请求会把成功账号数据发往外部服务 |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-08-01 | D | `cmd/api` uploader bootstrap | mitigate | 在 `cmd/api/main.go` 显式注入 `WithUploadAccountStore(accountsRepository)`，并用 bootstrap 级测试防止 route 挂载却运行时不可用。 |
| T-03-08-02 | T | `MarkSub2APIUploaded` writeback path | mitigate | 保持 03-04 已锁定的“仅成功账号写回”语义，不让失败/跳过账号被误标记上传成功。 |
| T-03-08-03 | I | upload response / secret fields | mitigate | 继续沿用 uploader handler/service 的兼容 DTO，list/upload 响应不泄露 api_key、token 或账号敏感字段。 |
</threat_model>
